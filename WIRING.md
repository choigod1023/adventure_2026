# adventure_2026 - 결선 가이드

본 문서는 [`config.h`](./config.h)의 핀 매핑에 따른 실제 결선 방법을 정리합니다.
변경할 일이 있으면 **반드시 `config.h` 와 같이** 갱신하세요.

| 핀 | 모듈 | 핀 이름 (모듈측) | 비고 |

## 부품별 결선 상세

### 1. OLED (SSD1306 128×64, I2C 4핀)

```
OLED        UNO R4
─────────────────────
VCC   ──→   3.3V (또는 5V)
GND   ──→   GND
SCL   ──→   A5
SDA   ──→   A4
```

- 모듈에 따라 핀 순서가 `GND-VCC-SCL-SDA` 또는 `VCC-GND-SCL-SDA` 일 수 있으니 **PCB에 인쇄된 표기 그대로** 연결.
- 화면이 안 켜지면 I2C 주소 (0x3C / 0x3D) 확인. U8g2의 NONAME 드라이버는 보통 0x3C 자동 인식.

### 2. HC-SR505 (PIR, 인체 감지)

```
HC-SR505    UNO R4
─────────────────────
VCC   ──→   5V
GND   ──→   GND
OUT   ──→   D2     (config.h: PIN_PIR)
```

- 부팅 후 **약 1분 안정화** 필요 (이 시간 동안 오감지 발생 가능).
- 감지 범위: 약 3m, 시야각 100°.
- HIGH 출력 유지 시간 ~2.5초 (정지하면 자동으로 LOW).

### 3. RCWL-0516 (마이크로파 도플러, 차량 감지)

```
RCWL-0516   UNO R4
─────────────────────
VIN   ──→   5V       (4~28V 입력 가능)
GND   ──→   GND
OUT   ──→   D3       (config.h: PIN_RADAR)
```

- 부팅 후 **약 1분 안정화** 필요 (PIR과 동일).
- 감지거리 기본 5~7m. 좁히려면 C-TM 핀에 저항 부착.
- **벽 투과 가능** → 사용자 본인 오감지 발생.
  → [`NOTES_RCWL_FALSEPOSITIVE.md`](./NOTES_RCWL_FALSEPOSITIVE.md) 의 PIR 게이팅 + 구리테이프 차폐 권장.

### 4. 모드 선택 버튼 4개 (택트 스위치)

각 버튼은 **핀 2개짜리 푸시버튼**으로 충분합니다.
코드가 `INPUT_PULLUP` 을 쓰므로 외부 풀업 저항 불필요.

```
버튼 (한쪽)  UNO R4              (반대쪽)
──────────────────────────────────────────
버튼1   ──→   D5  PIN_BTN_BUS     → GND
버튼2   ──→   D6  PIN_BTN_SUBWAY  → GND
버튼3   ──→   D7  PIN_BTN_SPAT    → GND
버튼4   ──→   D8  PIN_BTN_SENSOR  → GND
```

- 누르면 핀이 GND에 닿아 LOW. 안 누르면 내부 풀업으로 HIGH.
- 디바운스 20ms (`config.h: DEBOUNCE_MS`).

### 5. PAM8610 앰프 + 스피커 2개

```
PAM8610      UNO R4 / 전원 / 스피커
──────────────────────────────────────
VCC   ──→   ⚠️ DC 12V 별도 어댑터 (USB 5V로는 부족)
GND   ──→   GND (UNO R4와 공유)
IN_L  ──→   D9   ← 780Hz 톤 (PIN_SPK1_780HZ)
IN_R  ──→   D10  ← 2kHz 톤  (PIN_SPK2_2KHZ)
GND_L ──→   GND  (UNO R4 GND 공유)
GND_R ──→   GND  (UNO R4 GND 공유)
OUT_L ──→   스피커1 (+, −)
OUT_R ──→   스피커2 (+, −)
```

⚠️ **PAM8610은 DC 12V 전원 필수.** UNO R4의 5V로 구동 시도하지 말 것.
⚠️ **UNO R4와 GND 반드시 공유** — 안 그러면 톤이 안 나오거나 노이즈 발생.
- 볼륨 가변저항은 PAM8610 모듈에 내장되어 있어 별도 결선 불필요.

## 전원 정리

| 부품 | 전원 |
|---|---|
| UNO R4 WiFi | USB 5V (PC 또는 5V 어댑터) |
| PAM8610 | **별도 DC 12V 어댑터 (1~2A 권장)** |
| OLED / PIR / RCWL / 버튼 | UNO R4 의 5V 레일에서 공급 |

> 모든 GND는 **한 점에서 묶기** (star topology). 노이즈를 줄이고 톤 출력 안정성에 영향.

---

## 결선 순서 (디버깅하기 좋은 순서)

1. **UNO R4만 USB 연결** → 시리얼 모니터에서 부팅 로그 확인
2. **OLED I2C 연결** → 부팅 시 "부팅 중" 화면 표시되면 OK
3. **버튼 4개 (D5~D8)** → 누를 때마다 시리얼에 `[모드 전환] → BUS/SUBWAY/CITS/SENSOR` 로그
4. **PIR (D2)** → 시리얼에 `[PIR ↑]` 출력 확인 (1분 안정화 후)
5. **RCWL (D3)** → 시리얼에 `[RADAR ↑] #N` 출력 확인 (1분 안정화 후)
6. **PAM8610 + 스피커** → 12V 별도 전원 인가 → ④ 센서 모드에서 사용자 정지 + 움직이는 손으로 RCWL 트리거 → 780Hz/2kHz 교차 톤 확인

각 단계는 [`DEV_PROCESS.md`](./DEV_PROCESS.md) Phase 1 의 개별 테스트 스케치([`tests/`](./tests))로 사전 검증할 수 있습니다.

## 트러블슈팅

| 증상 | 원인/해결 |
|---|---|
| OLED 안 켜짐 | I2C 주소(0x3C/0x3D) 확인. SDA/SCL 바뀐 거 아닌지 확인. 3.3V/5V 둘 다 시도. |
| 버튼 눌러도 모드 안 바뀜 | 풀업 동작 확인. 시리얼에 로그 안 뜨면 GND 단자 연결 확인. 채터링은 `DEBOUNCE_MS` 조정. |
| RCWL이 계속 HIGH | 부팅 후 1분 안 지났거나, 주변 움직임 많음. 구리테이프 차폐 또는 R-GN 저항으로 감지거리 축소. |
| PIR 반응 늦음 | HC-SR505 자체 지연 ~2.5초. 정상. 다른 PIR 모듈로 교체하면 빠를 수 있음. |
| 톤 안 나옴 | PAM8610 12V 전원 인가됐는지. UNO R4와 GND 공유했는지. 위험 상태 아닐 때는 톤 없음(정상). |
| 톤만 노이즈 | 전원 GND 분리됨. star topology로 GND 통합. PAM8610 입력선과 전원선 꼬임 방지. |
